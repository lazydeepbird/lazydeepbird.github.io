'todo.txt 포맷(번역)'

작성일 2020-08-16 In productivity

GitHub - todotxt/todo.txt: ‼️ A complete primer on the whys and hows of todo.txt. 을 20-08-03에 번역하였다.

todo.txt의 사용과 방법에 대한 완전한 입문서.

todo.txt에 대한 첫번째이며 가장 중요한 규칙:

todo.txt 텍스트파일의 한 줄은 하나의 작업을 의미합니다.

왜 텍스트 파일일까요?

일반 텍스트 파일(Plain text)은 소프트웨어나 운영체제에 구애받지 않습니다. 검색가능하며 포터블하며 가볍고 쉽게 조작가능합니다. 구조화된 형태도 아닙니다. 누군가의 웹서버가 다운되었거나 Outlook의 .pst파일이 손상되었어도 작동합니다. 내보내기 및 가져오기도 없고, 데이터베이스도, 태그, 플래그, 별표, 우선순위 표시도 없고 어떤 것을 하거나 하지 않기 위해 특별한 규칙을 적용할 필요도 없습니다.

효과적인 할일 목록의 3가지 축

todo.txt의 특수 표기법을 이용해서 3개의 축으로 분할 가능한 목록을 만들 수 있습니다.

우선 순위(Priority)

할일 목록은 프로젝트이든 어떤 상황이든 수행해야 할 다음 중요한 일을 알려 줄 수 있어야 합니다. 할일을 선택적으로 우선순위를 부여함으로써 리스트의 위쪽으로 올릴 수 있습니다.

프로젝트(Project)

큰 프로젝트를 전진시키는 유일한 방법은 관련하여 작은 하위 작업을 만들고 진행하는 것입니다. 당신은 당신의 todo.txt에서 프로젝트의 모든 특정한 할일을 나열할 수 있어야 합니다.

컨텍스트(Context)

GTD의 창시자 데이비드 앨런(David Allen)은 작업 목록을 상황에 따라 - 작업할 장소와 상태 등 - 분할할 것을 제안합니다. 보낼 메시지는 @email로, 전화는 @phone로 가정의 업무는 @home으로 컨텍스트를 구분합니다.

이런식으로 하면 차에 타있는 몇분 동안 쉽게 @phone 작업을 확인하고 전화를 걸수 있습니다.

이런 것들은 모두 todo.txt에서 가능합니다.

todo.txt 포맷 규칙

todo.txt는 일반 텍스트 파일입니다. 우선순위, 프로젝트, 컨텍스트, 생성날짜, 완료날짜와 같은 구조화된 메타데이터를 활용하기 위한 단순하고 유연한 파일 형식 규칙이 있습니다.

철학적으로, todo.txt 파일 형식은 2개의 목표를 가지고 있습니다:

  • 파일 내용은 일반 텍스트 뷰어나 편집기 이외의 도구를 사용하지 않고도 사람이 읽을 수 있는 형태여야 합니다.
  • 사용자는 일반 텍스트 편집기에서 파일내용을 합리적이고 예상된 방식으로 수정할 수 있습니다. 예를 들어 알파벳 순으로 줄을 정렬할 수 있는 편집기는 작업 목록을 의미있는 방식으로 정렬 할 수 있어야 합니다.

이 두가지 목표는 우선순위 혹은 날짜로 시작하여 쉽게 정렬이 가능하고 x 표시로 완료된 아이템을 확인 가능하며 정렬 기능을 이용하여 완료 목록은 제일 아래에 위치할 수 있게 됩니다.

완료되지 않은 작업: 3가지 형식의 규칙

todo.txt의 장점은 완전히 구조화 되지 않았다는 겁니다. 각 작업에 첨부할 수 있는 필드는 상상력에 의해서만 제한됩니다.
컨텍스트(예:@phone), 프로젝트(예:+GarageSale), 우선순위(예:(A)) 등과 같은 특별한 기호표시를 사용할 수 있습니다.

규칙3. 컨텍스트와 프로젝트는 우선순위, 생성날짜 이후 줄의 아무곳에나 표기할 수 있습니다.

  • 컨텍스트는 빈한칸 다음에 @기호로 시작합니다.
  • 프로젝트는 빈한칸 다음에 +기호로 시작합니다.
  • 프로젝트와 컨텍스트는 사이에 빈칸이 있으면 안됩니다.
  • 하나의 할일 안에는 여러개의 프로젝트와 컨텍스트가 있을 수 있습니다.

완료된 할일: 2개의 포맷 규칙

규칙1. 완료된 작업은 소문자 x로 시작합니다.

만약 할일이 소문자 x로 시작하고 바로 다음에 빈칸(space)이 있다면, 완료로 표시된 것입니다.

완료된 작업은 표준 정렬 도구를 사용하여 작업목록의 제일 아래쪽으로 오도록 소문자 x를 사용합니다.

규칙2. 완료표시(x) 바로 뒤에 공백이 있어야 합니다.

예를 들면 아래와 같습니다.

생성날짜를 작업앞에 추가한 경우 완료 시 완료날짜 다음에 나타납니다. 완성 된 작업은 표준 정렬 도구를 사용하여 날짜 별로 정리됩니다. 많은 todo.txt클라이언트는 작업 완료 시 우선순위를 삭제합니다. 그것을 유지하기 위해 별도의 key:value(예를 들어 pri:A)를 사용하기도 합니다.

완료날짜(필수)와 생성날짜(선택)을 사용하는 경우 두 날짜의 차이를 이용하여 작업완료 기간을 계산할 수 있습니다.

추가 파일 형식 정의

툴의 개발자는 추가 메타데이터에 대한 형식을 정의할 수 있습니다.

예를 들어 개발자는 key:value 포맷을 사용하여 추가적인 메타데이터를 정의합니다(예: due:2010-01-21 형태를 구성하여 처리해야할 날짜 표시)

key와 value는 모두 공백없는 문자로 구성되어야 하며, 콜론(:)은 있어서는 안됩니다. key와 value를 구분하기 위한 하나의 콜론만 허용됩니다.
